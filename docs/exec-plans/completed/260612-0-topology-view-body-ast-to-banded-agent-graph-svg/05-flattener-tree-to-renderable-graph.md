All graph-shape policy lives here, in one reviewable place.

### Policy (locked)

- **Band mapping:** `phase` strings ‚Üí indices via `topology.bands` filtered to meta order first, body-only bands appended after (a body-only band still renders ‚Äî the renderer treats band list = meta.phases + appended extras; flattener emits the merged band-title list alongside or the renderer derives it ‚Äî concretely: flattener returns `{ir, bandTitles: string[]}` where the first `meta.phases.length` entries are meta's, extras appended; `TopoNode.band` indexes into that). `phase: null` steps ‚Üí band of the nearest following step with a band, else band 0 (note-free defensive default; corpus never hits it).
- **Sequence:** consecutive steps in the SAME band ‚Üí sequential edge. Implicit cross-band succession ‚Üí **NO edge** (vertical stacking already says it ‚Äî v1 semantics). Explicit structures crossing bands (pipeline stage chains, branch arms, loop arcs) DO get edges/loops.
- **`AgentStep`** ‚Üí one `TopoNode{kind:"agent"}`; multiplicity mapping: `one` ‚Üí `mult` absent; `exact`/`named`/`unknown` ‚Üí `FlatMultiplicity` (named carries `expandedLabels` as the names when present, else the raw names). `WorkflowStep`/`OpaqueStep` ‚Üí `TopoNode{kind:"task"}` (label as-is; opaque label reads as the honest blob).
- **`ParallelStep` fanout** ‚Üí one agent node carrying `mult` + one `barrier` node, edges node‚Üíbarrier; empty body (unresolvable arg) ‚Üí placeholder agent node labeled "agents", `mult` unknown ‚Üí barrier. **branches** ‚Üí k chains (each branch flattened in its own band placement), every chain's terminal node ‚Üí one shared `barrier`. The barrier sits in the band of the last contributing node.
- **`PipelineStep`** ‚Üí per-stage chains: each stage's steps flattened with the stage's lane multiplicity applied to its agent nodes (stage 1 gets `items` multiplicity); consecutive stages chained by an edge from stage k's terminal to stage k+1's head ‚Äî NO barrier between stages (pipeline semantics). Stages landing in different bands produce forward cross-band edges.
- **`BranchStep`** ‚Üí one `decision` node (label = conditionLabel); non-empty then-arm: edge decision‚Üíarm-head labeled **"yes"**, elseSteps likewise labeled **"no"**; arm chains flatten recursively; if a next step follows the branch in its sequence, each arm's terminal ‚Üí next (merge); an EMPTY arm with a following step ‚Üí direct decision‚Üínext edge (the bypass); branch as last step ‚Üí arms simply terminate.
- **`LoopStep`** ‚Üí flatten body chain, then synthesize one `decision` node after it (label = conditionLabel, banded at the loop's last body band) + `TopoLoop{from: decision, to: first body node, label: "yes"}` + (if a next step exists) edge decision‚Üínext labeled "no". This reproduces the Loop-Until-Done panel exactly. Nested loops nest naturally (inner decision is part of the outer body chain).
- `untaken` never set (v2). Node ids `n0, n1, ‚Ä¶` in emission order (determinism).
